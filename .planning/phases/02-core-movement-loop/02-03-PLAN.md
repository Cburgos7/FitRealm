<objective>
Build the passive-movement vertical slice: read HealthKit (iOS) / Health Connect (Android) daily distance + workouts, reconcile via gap-fill against already-credited active sessions, and bank the delta via an app-open prompt (credited to the Mile Bank only — never auto-feeds food). Front-load a physical-iPhone HealthKit smoke test as a Wave-0 risk gate for the react-native-health New-Architecture compatibility uncertainty (Assumption A1).

This plan is sequenced after Plan B (02-02) because (a) it adds the passive prompt + first-visit permission request into the same Move hub (apps/mobile/app/(tabs)/move/index.tsx) that Plan B builds out — keeping that file single-owner per wave — and (b) gap-fill READS the day_credits SQLite table that Plan B owns and writes on every GPS bank. This plan does NOT define dayCredits.ts; it imports getCreditedToday from @/lib/dayCredits (depends_on [02-01, 02-02] guarantees that file exists).

Purpose: Passive sync captures movement the GPS tracker misses (incidental walking, other apps' workouts) without double-counting.
Output: usePassiveHealth hook, gap-fill lib, app-open prompt, gapFill Wave-0 test green.
</objective>
