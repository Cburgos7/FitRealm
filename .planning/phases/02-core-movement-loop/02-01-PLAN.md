<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mobile client → Supabase (PostgREST/RPC) | Authenticated but untrusted; all writes must be RLS- and server-validated |
| npm/native package supply chain | New native packages execute build-time + runtime code |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Tampering | game_config table | mitigate | SELECT-only RLS policy (USING true); no client INSERT/UPDATE/DELETE policy — client can read but never alter balance |
| T-02-ID | Information Disclosure | villages / activities / allocations | mitigate | RLS owner scoping: villages USING auth.uid()=owner_id; activities/allocations USING auth.uid()=user_id |
| T-02-SC | Tampering | npm/native installs (Mapbox, health libs) | mitigate | RESEARCH Package Legitimacy Audit ([OK]) + blocking human checkpoint (Task 0); never auto-approved |
| T-02-DEC | Tampering | client decay attempt | accept→mitigate | No client decay code; decay is server-only (Plan E). Village screen reads food, never mutates it on a timer |
</threat_model>
