<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mobile client → Vercel API (/activity/manual) | Client-supplied distance/duration is untrusted |
| mobile client → Supabase Storage | Route file upload scoped to own folder |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02B-INF | Spoofing/Elevation | /activity/manual mile inflation | mitigate | Server derives miles from distance÷duration pace; rejects pace > manual_max_speed_mph (422); enforces manual_entry_daily_cap; never trusts client mileage |
| T-02B-GEO | Spoofing | fabricated GeoJSON route | mitigate | Credited miles come from the server-written activities row, not the GeoJSON file content (Security Domain) |
| T-02B-STO | Information Disclosure | routes bucket cross-user access | mitigate | storage.objects RLS scopes own-folder upload/read by auth.uid() (bucket policies from Plan A) |
| T-02B-SES | Tampering | replayed/duplicate session bank | mitigate | One activities insert per End→Bank; orphan checkpoint cleared after bank to prevent double-credit |
| T-02B-DUP | Tampering | GPS distance double-counted by passive gap-fill | mitigate | Each bank records its distance in the local day_credits table (dayCredits.addCredit); Plan C subtracts getCreditedToday from the health total (D2-06/Pitfall 4) |
</threat_model>
