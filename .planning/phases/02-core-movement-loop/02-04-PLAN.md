<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mobile client → allocate_food RPC | Client requests a spend; server must enforce balance + atomicity |
| SQLite outbox → Supabase (on sync) | Queued intents replayed; must be idempotent |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02D-RACE | Tampering | rapid-tap Hunt Food over-spend | mitigate | allocate_food SELECT ... FOR UPDATE locks the profile row; balance checked under lock (ALLOC-05) |
| T-02D-RPY | Tampering | replayed/duplicate allocation on sync | mitigate | idempotency_key UNIQUE in allocations + early-return in RPC + INSERT OR IGNORE in SQLite queue |
| T-02D-OVS | Spoofing/Elevation | client requests more food than miles | mitigate | RPC returns insufficient_miles when miles_banked < cost; client grey-out is only UX, server is authoritative (ALLOC-03) |
| T-02D-CFG | Tampering | client alters food rate/cap | mitigate | rate/cap read from game_config inside the RPC; game_config is SELECT-only RLS (Plan A) |
</threat_model>
